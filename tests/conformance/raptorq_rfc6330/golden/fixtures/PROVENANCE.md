# Golden File Provenance

This document tracks the provenance and generation methodology for RaptorQ RFC 6330 conformance golden files.

## Generation Information

- **Generated by**: asupersync conformance pipeline
- **Generator version**: v0.2.9
- **RFC version**: RFC 6330
- **Implementation**: asupersync RaptorQ encoder/decoder
- **Last updated**: 2026-04-16

## Generation Commands

### Parameter Derivation Fixtures
```bash
rch exec -- env CARGO_TARGET_DIR=${TMPDIR:-/tmp}/rch_target_raptorq_rfc6330_golden cargo run --manifest-path tests/conformance/raptorq_rfc6330/golden/Cargo.toml --bin generate_goldens -- --output fixtures/parameter_derivation --seed 42
```

## Test Coverage

### Parameter Derivation
- K calculation for various object sizes (1KB - 1MB)
- Systematic index derivation (J(K) from RFC Table 2)  
- Tuple generation parameters
- Sub-block structure parameters

### Constraint Matrices
- Systematic matrix structures (K=4 to K=8192)
- LDPC constraint matrices
- Half symbol matrices
- Matrix dimension validation

### Symbol Outputs
- Systematic symbol generation
- Repair symbol generation (ESI 0-99)
- Various data patterns (zeros, ones, random, repeating)
- Symbol format consistency

## Validation Methodology

1. **Manual verification**: Initial outputs manually verified against RFC 6330
2. **Cross-validation**: Compared against reference implementations where available
3. **Regression testing**: Frozen as golden files for ongoing validation
4. **Update process**: New fixtures require manual review and approval
